Первые шаги с APISafe

Шаг 1: Создание аккаунта

  1. Перейдите на apisafe.io
  2. Нажмите кнопку "Попробовать бесплатно"
  3. Заполните форму регистрации:
    • Введите ваш email адрес
    • Создайте надежный пароль (минимум 8 символов)
    • Согласитесь с условиями использования
  4. Подтвердите email через письмо
  5. Войдите в панель управления

Рекомендация по безопасности

Сразу после регистрации настройте двухфакторную аутентификацию в разделе "Настройки" → "Безопасность".

Шаг 2: Добавление первого API-ключа

  1. В панели управления нажмите кнопку + Добавить ключ
  2. Выберите тип сервиса из списка (например, "OpenAI API")
  3. Введите понятное название для ключа
  4. Вставьте ваш API-ключ в поле "Значение"
  5. Добавьте описание (опционально)
  6. Выберите проект или создайте новый
  7. Нажмите Сохранить

Готово!

Ваш API-ключ зашифрован и безопасно сохранен. Теперь вы можете использовать его в интеграциях.

Интеграция с Make.com

Подключение APISafe к Make.com

  1. В Make.com создайте новый сценарий
  2. Добавьте модуль и найдите "APISafe" в списке приложений
  3. Выберите действие "Get Secret Value"
  4. Нажмите "Create a connection"
  5. Введите ваш API токен APISafe:
    • Получите токен в APISafe: "Настройки" → "API токены"
    • Нажмите "Создать новый токен"
    • Скопируйте токен и вставьте в Make.com
  6. Нажмите "Save" для сохранения подключения

Использование секретов в сценариях

  1. В настройках модуля APISafe выберите нужный проект
  2. Выберите секрет из списка
  3. Запустите модуль для получения значения
  4. Используйте полученное значение в других модулях
// Пример использования в HTTP модуле Headers: Authorization: Bearer {{1.value}} // Где {{1.value}} - это значение из модуля APISafe

Командная работа

Приглашение участников

  1. Перейдите в раздел "Команда" в панели управления
  2. Нажмите кнопку "Пригласить участника"
  3. Введите email адрес коллеги
  4. Выберите роль:
    • Администратор - полный доступ к управлению
    • Разработчик - создание и использование ключей
    • Наблюдатель - только просмотр статистики
  5. Выберите проекты, к которым предоставить доступ
  6. Отправьте приглашение

Управление правами доступа

Для каждого участника можно настроить детальные права:

  • Просмотр секретов - видеть список и названия
  • Использование секретов - получать значения через API
  • Создание секретов - добавлять новые ключи
  • Редактирование секретов - изменять существующие
  • Удаление секретов - удалять ключи
  • Управление проектами - создавать и настраивать проекты

Настройка безопасности

Двухфакторная аутентификация (2FA)

  1. Перейдите в "Настройки" → "Безопасность"
  2. Нажмите "Включить 2FA"
  3. Отсканируйте QR-код приложением аутентификатора:
    • Google Authenticator
    • Authy
    • Microsoft Authenticator
  4. Введите код из приложения для подтверждения
  5. Сохраните резервные коды в безопасном месте

Управление сессиями

В разделе "Активные сессии" вы можете:

  • Просматривать все активные входы в аккаунт
  • Видеть информацию об устройствах и местоположении
  • Завершать подозрительные сессии
  • Настраивать автоматическое завершение сессий

Уведомления о безопасности

Настройте уведомления для получения информации о:

  • Входах с новых устройств
  • Изменениях в настройках безопасности
  • Подозрительной активности
  • Доступе к критичным секретам

Использование API

Получение API токена

  1. Войдите в панель управления APISafe
  2. Перейдите в "Настройки" → "API токены"
  3. Нажмите "Создать новый токен"
  4. Задайте название токена
  5. Выберите права доступа:
    • Чтение секретов
    • Создание секретов
    • Редактирование секретов
    • Удаление секретов
  6. Скопируйте токен (показывается только один раз)

Базовый пример использования

// Получение секрета через API curl -X GET "https://api.apisafe.io/v1/secrets/secret_abc123" \ -H "Authorization: Bearer YOUR_API_TOKEN" \ -H "Content-Type: application/json"

Использование SDK

// JavaScript/Node.js import { APISafe } from '@apisafe/sdk'; const client = new APISafe('your-api-token'); const secret = await client.getSecret('secret_abc123'); console.log(secret.value);

Аналитика и мониторинг

Просмотр статистики использования

В разделе "Аналитика" доступна информация о:

  • Общем количестве запросов к API-ключам
  • Самых используемых секретах
  • Активности по дням и часам
  • Источниках запросов (интеграции)
  • Ошибках и проблемах доступа

Настройка уведомлений

  1. Перейдите в "Настройки" → "Уведомления"
  2. Выберите события для уведомлений:
    • Превышение лимитов использования
    • Ошибки доступа к секретам
    • Подозрительная активность
    • Изменения в команде
  3. Настройте способы доставки:
    • Email уведомления
    • Webhook'и
    • Telegram бот

Устранение проблем

Частые проблемы и решения

Ошибка "Unauthorized" при API запросах

Причины:

  • Неверный или истекший API токен
  • Недостаточные права доступа
  • Неправильный формат заголовка Authorization

Решение:

  • Проверьте правильность токена
  • Убедитесь, что токен не истек
  • Проверьте права доступа токена
  • Используйте формат: Authorization: Bearer YOUR_TOKEN

Интеграция с Make.com не работает

Возможные причины:

  • Неправильно настроено подключение
  • Секрет не найден или удален
  • Превышен лимит API запросов

Решение:

  • Пересоздайте подключение в Make.com
  • Проверьте существование секрета в APISafe
  • Проверьте лимиты в разделе "Использование"

Не могу войти в аккаунт

Если забыли пароль:

  • Нажмите "Забыли пароль?" на странице входа
  • Введите ваш email
  • Перейдите по ссылке в письме
  • Создайте новый пароль

Если проблемы с 2FA:

  • Используйте резервные коды
  • Обратитесь в поддержку для сброса 2FA
